# 6529 NEWS - Project Guide

## Overview
Decentralized news platform for the 6529 NFT ecosystem. Two main products:
- **News Engine** (6529news_engine): Live news cards, ticker, breaking news
- **Governance** (6529news_gov): TDH-weighted proposal voting system

## Architecture

### News Engine (`index.html` + `generate-news.py`)

**Single-page HTML** with inline CSS/JS. Three news sections:

#### 1. Main News (rotating cards, 8s each)
Cards generated by `generate-news.py`, stored in `data/news-latest.json`:
- **MINTING STATUS** ‚Äî 4 states: MINTING TODAY / MINTING [DAY] / STILL MINTING / NEXT DROP. Schedule: selections Mon/Wed/Fri at 16:00 UTC, mint 2 days later (Fri‚ÜíMon = 3 days). Category badge matches headline day name.
- **TOP MEMES** ‚Äî Top 3 by projected TDH, preview = #1's image. Shows exact next selection datetime (not countdown).
- **NEW MEMES SUBMISSIONS** ‚Äî "X New Memes Submissions This Week". Last 7 days, excludes top 3 leaderboard authors, filters out chat messages (title=None/empty). Uses `/api/waves/{id}/drops` endpoint.
- **TOP SUPERRARE** ‚Äî Top 3 databoxes, random preview from top 20 (changes each refresh).
- **SALES RECAP** ‚Äî 24h sales from OpenSea, filtered by event_timestamp.
- **6529 TALKS** (conditional) ‚Äî Appears when punk6529 has 5+ msgs in 24h. Shows total msg count with paginated fetch. AI summary: 5-8 word topic (max_tokens=30).

Card rendering: `buildCards()` function. Layout: preview fills space (`flex: 1`), text pushed to bottom with `.card-text { margin-top: auto }`.

HTML submissions (like ikertje's): check `metadata.additional_media.preview_image` for preview in BOTH `fetch_ranked_drops()` AND `_parse_winner()`. Validate with HEAD request (CDN may 403).

#### 2. Breaking News (one at a time, rotates each card change, label "NEWS")
Headlines rotate one per card change (8s). If text overflows, scrolls with animation.

**Fixed headlines** (always present):
- **PUNK6529** ‚Äî Continuity logic:
  - Active now (last 1h): `PUNK6529 ACTIVE NOW IN [WAVE] WITH XXX MSGS`
  - Active today: `PUNK6529 BACK TODAY IN [WAVE] WITH XXX MSGS`
  - Yesterday/2d ago: `PUNK6529 BACK YESTERDAY IN [WAVE] WITH XXX MSGS`
  - 3+ days inactive: `PUNK6529 LAST SEEN: [wave] ([date])`
  - Message count uses full pagination (not just first 20)
- **PEBBLES**: floor + sales count + last sale time
- **DIVE BAR**: `DIVE BAR: X MSGS IN LAST 6H` ‚Äî uses `/api/waves/{id}/drops` with pagination

**Conditional headlines**:
- SWEEP ALERT (5+ sales of same card)
- NAKA SALE (Nakamoto card sale in last hour from OpenSea)

#### 3. Stats Ticker (continuous scroll bar, label "STATS")
Network TDH, Full Set Bid, Holders, Full Set, #1 artist, 24h/7d Vol.
Each stat has `raw` numeric value for delta calculation. Delta shown in parentheses:
- Green `#4eca6a` for positive, Red `#e85454` for negative
- Units match the stat: `+1.2M TDH`, `+0.30 ETH`, `+12` (plain)
- Delta calculated vs previous JSON run

### Data Generation & Refresh (every 15 min)

**`generate-news.py`** ‚Äî Main generator.
- APIs: 6529.io (waves, drops, leaderboard), OpenSea (sales, floor)
- AI: OpenRouter (Nemotron) for punk6529 topic summaries
- **IMPORTANT**: always use `/api/waves/{id}/drops` endpoint (not `/api/drops?wave_id=`). The old endpoint returns incomplete results.
- API limit=20 max per page (higher values silently cap to 10). Always paginate.
- punk6529 messages: paginate ALL drops in 24h (can be 100+), not just first page
- SuperRare disabled: `build_sr_submissions()` commented out, `build_top_superrare()` active

**`check-6529.yml`** ‚Äî Every 15 min. Updates punk6529 headlines + dive bar count + 6529 TALKS card in existing JSON. Calls `build_punk6529()` and `build_divebar_activity()` from generate-news.py.

**`generate-news.yml`** ‚Äî Every 15 min (`*/15 * * * *`) + Mon/Wed/Fri 16:01 UTC (post-selection).

**Client refresh**: `setInterval(loadDynamicNews, 15*60*1000)` in index.html.

### Intro Animation
Word wall background: giant "Anton" font words (TDH/MEMES/BRAIN/ART) in rows, filled letters in gold transparent (`rgba(200,168,78,0.05)`). Pure CSS, no JS.

### Governance (`governance/`)

**SPA with hash routing**. Files: `js/app.js`, `js/wallet.js`, `js/proposals.js`, `js/voting.js`, `js/api6529.js`, `js/config.js`, `css/styles.css`.

**Voting system**:
- Proposals/votes stored as GitHub Issues on `6529news_gov`
- EIP-712 signed with wallet, submitted via Cloudflare Worker proxy
- TDH-weighted: bipolar slider from -MAX to +MAX TDH, magnetic snap to zero (5% zone)
- Manual TDH input field synced with slider
- Threshold: 30M TDH to pass, 1M TDH minimum to propose
- Duration: 30 days
- **GitHub as TDH source of truth** ‚Äî `tallyVotes()` reads `allocatedTDH` from vote issues, NO live API calls to 6529

**Cloudflare Worker** (`worker/src/index.js`):
- POST `/api/proposal` ‚Äî create proposal issue
- POST `/api/vote` ‚Äî create vote issue
- POST `/api/close-issue` ‚Äî withdraw vote/delete proposal
- GET `/api/issues` ‚Äî proxy GitHub Issues API (avoids 60/hr rate limit)
- Deployed at: `6529news-gov.24h6529.workers.dev`

**Key frontend features**:
- Wallet persists across refresh (localStorage + `eth_accounts` silent reconnect)
- Auto-switch to Ethereum Mainnet before EIP-712 signing
- Dashboard: progress bar + vote count + TDH tally on each proposal card (single API call for all votes)
- Voted proposals highlighted with green "‚úì VOTED" badge
- Change vote: withdraw old + submit new in one flow
- User area (top-right) shows free TDH (green) + allocated TDH (gold), updates after vote/withdraw (4s delay for GitHub propagation)
- Confirm button disabled until slider moves from initial position

## Known Issues & Gotchas
- **API endpoint**: `/api/drops?wave_id=` returns INCOMPLETE results ‚Äî ALWAYS use `/api/waves/{id}/drops`
- **API pagination**: limit=20 max per page. limit=50 silently returns 10. Always paginate with `serial_no_less_than`.
- **CDN 403**: `preview_image` for HTML submissions may return 403. Always validate with HEAD request before using.
- **GitHub rate limit**: unauthenticated = 60/hr. ALL frontend GitHub API calls go through Worker proxy (5000/hr).
- **check-6529.yml overwrites**: This workflow modifies `headline_extras` in the JSON. It removes old PUNK6529/DIVE BAR entries and re-adds them. Must stay in sync with generate-news.py headline logic.
- **Fallback sync**: `update-fallback.py` updates hardcoded fallback data in index.html on each generate run.
- **punk6529 msg count**: Must paginate ALL drops (can be 100+/day). First page only gives 20.
- **Two workflows can conflict**: generate-news and check-6529 both modify `news-latest.json`. If they run simultaneously, one push will be rejected. The cron offsets help but GitHub Actions timing is approximate.
