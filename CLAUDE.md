# 6529 NEWS - Project Guide

## Overview
Decentralized news platform for the 6529 NFT ecosystem. Two main products:
- **News Engine** (6529news_engine): Live news cards, ticker, breaking news
- **Governance** (6529news_gov): TDH-weighted proposal voting system

## Architecture

### News Engine (`index.html` + `generate-news.py`)

**Single-page HTML** with inline CSS/JS. Three news sections:

#### 1. Main News (rotating cards, 8s each)
Cards generated by `generate-news.py`, stored in `data/news-latest.json`:
- **MINTING TODAY/TOMORROW/WEDNESDAY/STILL MINTING** — 4 states based on mint schedule (Mon/Wed/Fri selections at 16:00 UTC, mint 2 days later)
- **TOP MEMES** — Top 3 by projected TDH, preview = #1's image
- **NEW MEMES SUBMISSIONS** — Last 7 days, excludes top 3 authors, filters out chat messages (title=None)
- **TOP SUPERRARE** — Top 3 databoxes, random preview from top 20
- **SALES RECAP** — 24h sales from OpenSea, filtered by timestamp

Card rendering: `buildCards()` function. Layout classes: `.card-has-data`, `.card-no-data`, `.card-big-preview`, `.card-multi-preview`, `.card-no-image`. Text block pushed to bottom with `.card-text { margin-top: auto }`.

#### 2. Breaking News (continuous scroll bar, label "NEWS")
All headlines concatenated and scroll continuously (not alternating).
**Fixed headlines** (always present):
- PUNK6529 LAST SEEN / ACTIVE NOW
- PEBBLES FLOOR + sales + last sale time
- DIVE BAR: X MSGS IN LAST 6H

#### 3. Stats Ticker (continuous scroll bar, label "STATS")
Network TDH, Full Set Bid, Holders, Full Set, #1 artist, 24h/7d Vol.
Each stat has `raw` value for delta calculation. Delta shown in green/red with matching unit (M TDH, ETH, plain number).

### Data Generation

**`generate-news.py`** — Main generator, runs every 15 min via GitHub Actions.
- APIs: 6529.io (waves, drops, leaderboard), OpenSea (sales, floor)
- AI: OpenRouter (Nemotron) for punk6529 summaries
- Important: use `/api/waves/{id}/drops` endpoint (not `/api/drops?wave_id=`), limit=20 max per page
- HTML submissions: check `metadata.additional_media.preview_image` for preview
- Preview validation: HEAD request to verify CDN URLs aren't 403

**`check-6529.yml`** — Every 15 min, updates punk6529 + dive bar headlines in existing JSON.

**`generate-news.yml`** — Every 15 min + Mon/Wed/Fri 16:01 UTC (post-selection).

### Governance (`governance/`)

**SPA with hash routing**. Files: `js/app.js`, `js/wallet.js`, `js/proposals.js`, `js/voting.js`, `js/api6529.js`, `js/config.js`, `css/styles.css`.

**Voting system**:
- Proposals/votes stored as GitHub Issues on `6529news_gov`
- EIP-712 signed with wallet, submitted via Cloudflare Worker proxy
- TDH-weighted: allocate positive or negative TDH (bipolar slider)
- Threshold: 30M TDH to pass, 1M TDH minimum to propose
- Duration: 30 days
- GitHub as TDH source of truth (no live API calls for tally)

**Cloudflare Worker** (`worker/src/index.js`):
- POST `/api/proposal` — create proposal issue
- POST `/api/vote` — create vote issue
- POST `/api/close-issue` — withdraw vote/delete proposal
- GET `/api/issues` — proxy GitHub Issues API (avoids rate limit)
- Deployed at: `6529news-gov.24h6529.workers.dev`

**Key features**:
- Wallet persists across refresh (localStorage)
- Voted proposals highlighted with green badge
- Bipolar slider: -MAX to +MAX TDH, magnetic snap to zero
- Manual TDH input synced with slider
- Change vote = withdraw old + submit new
- User area updates after vote/withdraw

## Push Workflow
```bash
# Engine (news + generate scripts)
git add <files> && git commit -m "msg" && git pull engine main --no-rebase && git push engine main

# Governance (frontend only — copy to cloned repo)
cp governance/js/*.js /tmp/6529news_gov/js/
cp governance/css/*.css /tmp/6529news_gov/css/
cd /tmp/6529news_gov && git add -A && git commit -m "msg" && git push origin main

# Trigger workflows
gh api repos/6529news/6529news_engine/actions/workflows/251225286/dispatches -X POST -f ref=main  # Generate News
gh api repos/6529news/6529news_engine/actions/workflows/251358007/dispatches -X POST -f ref=main  # punk6529 check

# Deploy worker
cd worker && npx wrangler deploy
```

## Known Issues & Gotchas
- 6529 API `/api/drops?wave_id=` returns incomplete results — always use `/api/waves/{id}/drops`
- API limit=20 max per page (higher values silently cap to 10)
- CDN preview_image for HTML submissions may return 403 — validate with HEAD request
- GitHub unauthenticated API rate limit: 60/hr — all frontend calls go through Worker proxy
- check-6529.yml can overwrite headlines from generate-news — both must manage the same headline keys
- `update-fallback.py` updates hardcoded fallback data in index.html on each generate run
