# 6529 NEWS - Project Guide

## Overview
Decentralized news platform for the 6529 NFT ecosystem. Two main products:
- **News Engine** (6529news_engine): Live news cards, ticker, breaking news
- **Governance** (6529news_gov): TDH-weighted proposal voting system

## Architecture

### News Engine (`index.html` + `generate-news.py`)

**Single-page HTML** with inline CSS/JS. Three news sections:

#### 1. Main News (rotating cards, 8s each)
Cards generated by `generate-news.py`, stored in `data/news-latest.json`.

**Current card order:**
1. **MINTING STATUS** — 4 states: MINTING TODAY / MINTING [DAY] / STILL MINTING / NEXT DROP. Schedule: selections Mon/Wed/Fri at 16:00 UTC, mint 2 days later (Fri→Mon = 3 days). Category badge matches headline day name.
2. **TOP MEMES** — Top 3 by projected TDH, preview = #1's image. Shows exact next selection datetime.
3. **NEW WAVE** (temporary, expires Apr 10) — Network Museum SAFE Signers announcement. Preview: card #349 "Wanderer of a New Era".
4. **MUSEUM SIGNERS** (temporary, expires Apr 10) — Leaderboard top 3 candidates with databoxes. Preview: card #349. Updates every 15 min.
5. **NEW MEMES SUBMISSIONS** — "X New Memes Submissions This Week". Last 7 days, excludes top 3 leaderboard authors, filters out chat messages (title=None/empty).
6. **TOP SUPERRARE** — Top 3 databoxes, random preview from top 20.
7. **6529 TALKS** (conditional, 5+ msgs in 24h) — punk6529 activity. If ≤42 msgs: just "Active in [wave]". If >42: shows count. No AI summary.
8. **SALES RECAP** — 24h sales from OpenSea, filtered by event_timestamp.

Card rendering: `buildCards()` function. Layout: preview fills space (`flex: 1`), text pushed to bottom with `.card-text { margin-top: auto }`.

HTML submissions (like ikertje's): check `metadata.additional_media.preview_image` for preview in BOTH `fetch_ranked_drops()` AND `_parse_winner()`. Validate with HEAD request (CDN may 403).

#### 2. Breaking News (one at a time, rotates each card change, label "NEWS")
Headlines rotate one per card change (8s). If text overflows, scrolls with animation.

**Fixed headlines** (always present):
- **PUNK6529** — Continuity logic:
  - Active now (last 1h): `PUNK6529 ACTIVE NOW IN [WAVE] WITH XXX MSGS`
  - Active today: `PUNK6529 BACK TODAY IN [WAVE] WITH XXX MSGS`
  - Yesterday/2d ago: `PUNK6529 BACK YESTERDAY IN [WAVE] WITH XXX MSGS`
  - 3+ days inactive: `PUNK6529 LAST SEEN: [wave] ([date])`
  - Message count uses full pagination (can be 100+/day)
- **DIVE BAR**: `DIVE BAR: X MSGS IN LAST 6H`
- **NEW WAVE** (temporary, expires Apr 10): `NEW WAVE: NETWORK MUSEUM SAFE SIGNERS — APPLY NOW`

**Conditional headlines**:
- PEBBLES: floor + sales + last sale (only shown when sales > 0)
- SWEEP ALERT (10+ sales of same card in 1h)
- NAKA/PEPE/EXIT STRATEGY sale headlines (24h window)
- Volume % change

#### 3. Stats Ticker (continuous scroll bar, label "STATS")
Network TDH, Full Set Bid, Holders, Full Set, #1 artist, 24h/7d Vol.
Each stat has `raw` numeric value + delta vs previous run:
- Green `#4eca6a` for positive, Red `#e85454` for negative
- Units match the stat: `+1.2M TDH`, `+0.30 ETH`, `+12`
- History saved in `data/ticker-history.json`

### Data Generation & Refresh (every 15 min)

**`generate-news.py`** — Main generator.
- APIs: 6529.io (waves, drops, leaderboard), OpenSea (sales, floor)
- AI: OpenRouter (Nemotron) — currently unused (punk6529 summary removed)
- **CRITICAL**: always use `/api/waves/{id}/drops` endpoint (not `/api/drops?wave_id=`). The old endpoint returns incomplete results.
- API limit=20 max per page (higher values silently cap to 10). Always paginate with `serial_no_less_than`.
- punk6529 messages: paginate ALL drops in 24h (can be 100+)
- Museum wave: `a2ed7791-6402-4333-9780-d7af1fdce918`, expires `MUSEUM_EXPIRES = datetime(2026, 4, 10, ...)`

**`check-6529.yml`** — Every 15 min. Updates punk6529 headlines + dive bar count + 6529 TALKS card in existing JSON. Removes only PUNK6529/DIVE BAR headlines, preserves all others.

**`generate-news.yml`** — Every 15 min (`*/15 * * * *`) + Mon/Wed/Fri 16:01 UTC (post-selection).

**Client refresh**: `setInterval(loadDynamicNews, 15*60*1000)` in index.html.

### Intro Animation
Word wall background: giant "Anton" font words (TDH/MEMES/BRAIN/ART) in rows, filled letters in gold transparent (`rgba(200,168,78,0.05)`). Pure CSS, no JS.

### Governance (`governance/`)

**SPA with hash routing**. Files: `js/app.js`, `js/wallet.js`, `js/proposals.js`, `js/voting.js`, `js/api6529.js`, `js/config.js`, `css/styles.css`.

**Voting system**:
- Proposals/votes stored as GitHub Issues on `6529news_gov`
- EIP-712 signed with wallet, submitted via Cloudflare Worker proxy
- TDH-weighted: bipolar slider from -MAX to +MAX TDH, magnetic snap to zero (5% zone)
- Manual TDH input field synced with slider
- Threshold: 30M TDH to pass, 1M TDH minimum to propose
- Duration: 30 days
- **GitHub as TDH source of truth** — `tallyVotes()` reads `allocatedTDH` from vote issues, NO live API calls

**Cloudflare Worker** (`worker/src/index.js`):
- POST `/api/proposal` — create proposal issue
- POST `/api/vote` — create vote issue
- POST `/api/close-issue` — withdraw vote/delete proposal
- GET `/api/issues` — proxy GitHub Issues API (avoids 60/hr rate limit)
- Deployed at: `6529news-gov.24h6529.workers.dev`

**Key frontend features**:
- Wallet persists across refresh (localStorage + `eth_accounts` silent reconnect)
- Auto-switch to Ethereum Mainnet before EIP-712 signing
- Dashboard: progress bar + vote count + TDH tally on each proposal card
- Voted proposals highlighted with green "✓ VOTED" badge
- Change vote: withdraw old + submit new in one flow
- User area (top-right): free TDH (green) + allocated TDH (gold), updates after vote (4s delay)

## Push Workflow
```bash
# Engine (news + generate scripts)
git add <files> && git commit -m "msg" && git pull engine main --no-rebase && git push engine main

# Governance (frontend only — copy to cloned repo, then push)
# First time: git clone https://...6529news_gov.git /tmp/6529news_gov
cp governance/js/*.js /tmp/6529news_gov/js/
cp governance/css/*.css /tmp/6529news_gov/css/
cd /tmp/6529news_gov && git add -A && git commit -m "msg" && git push origin main

# Trigger workflows
gh api repos/6529news/6529news_engine/actions/workflows/251225286/dispatches -X POST -f ref=main  # Generate News

# Deploy Cloudflare Worker
cd worker && npx wrangler deploy
```

## Known Issues & Gotchas
- **API endpoint**: `/api/drops?wave_id=` returns INCOMPLETE results — ALWAYS use `/api/waves/{id}/drops`
- **API pagination**: limit=20 max per page. limit=50 silently returns 10. Always paginate with `serial_no_less_than`.
- **CDN 403**: preview_image for HTML submissions and wave pictures often return 403 from external. Validate with HEAD request. PFP images on IPFS gateways are slow/unreliable.
- **GitHub rate limit**: unauthenticated = 60/hr. ALL frontend GitHub API calls go through Worker proxy (5000/hr).
- **check-6529.yml overwrites**: Modifies `headline_extras` in JSON. Removes PUNK6529/DIVE BAR entries and re-adds. Must NOT touch other headlines (NEW WAVE, PEBBLES, etc).
- **Two workflows conflict**: generate-news and check-6529 both modify `news-latest.json`. If they run simultaneously, one push will be rejected. Cron offsets help but GitHub timing is approximate.
- **punk6529 msg count**: Must paginate ALL drops (can be 100+/day). First page only gives 20.
- **Fallback sync**: `update-fallback.py` updates hardcoded fallback data in index.html on each generate run.
- **Museum wave temporary**: Cards and headline expire Apr 10 (`MUSEUM_EXPIRES`). Remove from check-6529.yml headline filter when expired.
